Le DSL de message

Partout où KDA produit un message — que tu l'envoies toi-même, que tu répondes à un événement ou à une interaction — tu passes le même DSL de construction. Apprends-le une fois, réutilise-le partout.

Ce topic couvre le corps du message (texte, options, réponses éphémères). Les deux briques riches ont leur propre référence : Embeds et Boutons.

Où s'ouvre le DSL

Toutes ces fonctions prennent un bloc ResponseScope.() -> Unit — c'est le DSL décrit ici :

Fonction

Contexte

Effet

sendMessage(channelId) { }

client

envoie un message dans un salon

reply { }

événement message

répond dans le salon du message reçu

respond { }

interaction

répond à une commande ou à un composant

update { }

interaction de composant

édite le message porteur du composant

editOriginal { }

interaction

édite la réponse déjà envoyée (après defer)

defer() (sans bloc) accuse réception sans corps ; tu complètes ensuite avec editOriginal { }. Voir Réponses aux interactions.

Le corps du message

Dans le bloc, tu règles directement les champs du message :

Champ

Type

Défaut

Rôle

content

String?

null

le texte du message

tts

Boolean

false

lecture text-to-speech

flags

Int?

null

bitfield brut de flags (voir ephemeral())

import ktordiscord.components.snowflake val channelId = "123456789012345678".snowflake kda.sendMessage(channelId) { content = "Un message texte simple." tts = false }

À cela s'ajoutent deux briques qui s'attachent via des sous-blocs plutôt que par affectation :

  • embed { } — ajoute un embed (autant que tu veux). → Référence des embeds

  • button(...) { } — ajoute un bouton, rangé automatiquement en action row. → Référence des boutons

Il n'y a pas de setter embeds = ni components =: ces collections se remplissent uniquement via embed { } et button(...).

Réponses éphémères

Dans une réponse d'interaction, ephemeral() marque le message comme visible du seul utilisateur qui a déclenché l'interaction :

kda.on("secret") { respond { ephemeral() // Seul l'utilisateur qui a lancé la commande voit ce message. content = "Ceci n'est visible que par toi." } }

En interne, ephemeral() positionne le flag 1 shl 6 dans flags. C'est le seul flag exposé par un helper ; tout autre flag se règle à la main via flags.

ephemeral() n'a de sens que sur une réponse d'interaction (respond/update). Sur un sendMessage classique, Discord l'ignore.

Récupérer le message créé

sendMessage et reply renvoient un DiscordResponse que tu peux dérouler :

import ktordiscord.components.snowflake val channelId = "123456789012345678".snowflake // Récupérer l'objet, ou null en cas d'échec val message = kda.sendMessage(channelId) { content = "Hello!" }.getOrNull() // Réagir uniquement au succès et/ou à l'échec (chaînable) kda.sendMessage(channelId) { content = "Hello!" } .onSuccess { println("id = ${it.id}") } .onFailure { println("échec ${it.status} : ${it.error?.message}") } // Récupérer l'objet, ou lever une DiscordApiException en cas d'échec val created = kda.sendMessage(channelId) { content = "Hello!" }.getOrThrow()
